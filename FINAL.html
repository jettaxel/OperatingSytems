Domain Users

About this Video

- Add domain users (distribute IT student names to all dept/section/area)
- Create domain users for each department/office and technology section. use their real names when creating the domain account
- Create an organizational unit for each technology section or office (i.e. computer engineering, electronics, registrar, research etc.)
- All users can only log on to workstations that were assigned to their respective departments/offices
- All users should change their passwords every 30 days
- Academic users can log on from 7:30 am - 8:45am while administrative staff users can log on from 7:00am - 7:00pm

DHCP

About this Video

I. IP Block 172.31.0.0/22
II. router ip address is valid IP address
III. DNS server IP address is the same as the DHCP server. also add google and opendns public dns server ip addresses.
IV. ip address lease expires every 8 hrs.
V. do not distribute the first 30 and last 30 valid IP addresses.

File Services

About this Video

I. All departments/sections should have one public folder (read/write). domain users cannot access other department folders.
II. a university public folder where all domain users can access (read/write)
III. an installer folder containing applications that only the administrator can access.
IV. all folders except the computer engineering folder should contain only office documents (.ppt, .doc, .docx, .xls, .pptx, .pdf etc.)
V. information technology folder should contain all files except executable files.
VI. the storage quota for each folder is 1GB.
VII. map the shared folder for each client workstation.
